# Semester Backup Tool

A reusable macOS script that backs up your school files to an external SSD every semester.

## Features

- Scans your whole home directory for school files
- Matches names containing: `comp` + course number, `project`, `assignment`, `lab`, `midterm`, `term-proj`, `final-report`, `weekly-progress`, `seneca`
- Filters by a configurable date range (set per semester)
- Preserves your original folder structure inside the backup
- Auto-syncs to your external SSD when it's plugged in
- `--cleanup` mode deletes local originals **only after** the SSD sync is confirmed
- Writes a full log of everything copied, skipped, and deleted
- **Optional `--setup`**: name the backup folder, add your course codes, and add extra project names — all on top of the existing filters

---

## Usage

### Run the backup (uses saved config or built-in defaults)
```bash
bash collect_semester.sh
```

### Backup + delete local files to free space
```bash
bash collect_semester.sh --cleanup
```

### Configure a new semester (optional but recommended)
```bash
bash collect_semester.sh --setup
```

### To Cancel the setup prompts

```Ctrl + C
```

---

## Setup wizard

Run `--setup` once at the start of each semester. It asks:

| Prompt | Example |
|---|---|
| Semester label | `Y3S1` |
| Start date | `2026-09-01` |
| End date | `2026-12-31` |
| SSD name | `X10 Pro` *(Enter to keep)* |
| Course codes | `COMP3044 COMP3087 COMP3152` |
| Extra names | `afterthebeep my-portfolio` |

Saves everything to `semester.conf`. If you skip `--setup`, the script runs with its built-in defaults (Y2S2 settings).

---

## How course codes work

The script already finds anything matching `comp` + a number (e.g., `comp2139`, `COMP3044`).

Adding course codes in `--setup` lets you search by **full code name**, which catches files like:
- `COMP3044-lab1.py`
- `comp3044_assignment2.pdf`
- `COMP3044 Final Report.docx`

Extra names work the same way — type `afterthebeep` and any file or folder with that word in the name gets included.

---

## semester.conf

Auto-generated by `--setup`. You can also edit it directly:

```
SEMESTER_LABEL="Y3S1"
START_DATE="2026-09-01"
END_DATE="2026-12-31"
SSD_NAME="X10 Pro"
COURSES="COMP3044 COMP3087 COMP3152"
EXTRA_NAMES="afterthebeep my-portfolio"
```

Delete `semester.conf` to go back to built-in defaults.

## What is always excluded

`node_modules` `.git` `bin` `obj` `__pycache__`
`.cursor` `.lmstudio` `.nuget` `.platformio` `.ServiceHub` `.dotnet` `.local` `.cache`

## Manual sync (if SSD wasn't plugged in)

```bash
rsync -av --progress ~/<LABEL>_Semester_Backup/ "/Volumes/X10 Pro/<LABEL>_Semester_Backup/"
```
